### Deferred

- **Full `api-spec.md` regeneration** from router source. The current
  spec covers the core REST API faithfully; the freshness note at top
  flags the additions. A real regeneration wants a small Go tool that
  walks chi routes + JSON-Schema-extracts the request/response types —
  out of scope for the audit pass.
- **`kubernetes-service-discovery.md`, `api-deprecation-strategy.md`,
  `zero-downtime-deployment.md` deep validation.** Spot-checked each;
  they're reference docs for hypothetical operator scenarios and
  remain plausible. Will only need a refresh once those scenarios
  become real (k8s deployment, first deprecation, first zero-downtime
  cut-over).
- **ADR `superseded by` cross-references.** All 23 ADRs are accurate;
  the only candidate (ADR-007 → ADR-023 for runtime heartbeat retry)
  is a "see also" pointer, not a true supersession. Skipped to keep
  ADRs immutable as a matter of principle.

### What this gives you

A user installing RAT today can:
1. Read `README.md` / `CLAUDE.md` and understand the architecture and how to extend it.
2. Find every public env var with current values + security notes in `docs/config.md`.
3. Install any plugin via copy-paste `docker run` or the compose overlay (per-plugin READMEs + `infra/docker-compose.plugins.yml`).
4. Upgrade from any pre-Wave-8 snapshot using `docs/migrations/`.
5. Get an honest picture of what's documented vs what's behind reality on the API surface.
