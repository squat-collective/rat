# Python rules (runner + query)

**Toolchain:** Python 3.12 · `uv` (package mgr) · `ruff` **pinned 0.6.9** · `pyright` strict · `pytest` + `pytest-cov` · DuckDB · PyArrow · PyIceberg.

## Style
- Type hints everywhere; `pyright` strict — no `# type: ignore` (fix the type).
- Explicit error handling, no bare `except`; `raise X(...) from e`.
- `@dataclass(frozen=True)` / Pydantic for structured data; no mutable globals.
- **No Python loops for data manipulation** — use DuckDB SQL or PyArrow.
- `isinstance(x, A | B)` not `(A, B)`; `dict.get(k, default)` not ternary (ruff UP038/SIM401).

## ruff — the gotchas that bit us
- **Pinned to 0.6.9** in the Makefile + both `pyproject.toml` dev deps + `ci.yml`'s `pipx install ruff==0.6.9`. Unpinned `ruff` drifts and breaks green builds; never loosen the pin without re-validating.
- pyproject uses **`extend-exclude`** (not `exclude`) so ruff's built-in excludes (`.venv`, `.ruff_cache`, `__pycache__`) still apply. If you see "Found 70000+ errors," ruff is scanning a local `.venv` — that means the exclude got clobbered.
- The lint config uses the **`TCH`** selector name (not `TC`) — 0.6.9 can't parse `TC001`.
- `N802` is silenced for stdlib-API-mandated method names: `visit_Attribute`/`visit_Subscript` (ast.NodeVisitor) via inline `# noqa: N802`; `do_POST` (http.server) via the `tests/**` per-file-ignore.

## protobuf gencode
The Python `*_pb2.py` are generated by the buf plugin **pinned to v33** in `proto/buf.gen.yaml` so gencode targets the 6.33.x runtime that `uv` installs. Unpinned → gencode targets 7.x → `VersionError` at import. Bump the pin in lockstep with the runtime.

## Pipeline trust model (ADR-017)
Python pipelines (`pipeline.py` via `python_exec.py`) are **second-party trusted** — operator-written, loaded from S3. The runner container hardening (`read_only`, `cap_drop: ALL`, `no-new-privileges`) is the real boundary; the `exec()` blocklist is defense-in-depth only. Never expose a "submit arbitrary Python" API; never rely on the blocklist for new threat models.

## Before pushing
`make ci` runs runner (611) + query (202) suites. The `git push` hook covers lint; run `make ci` yourself when you touch Python tests.
