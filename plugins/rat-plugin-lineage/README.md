# rat-plugin-lineage

Pipeline lineage DAG viewer. Reads ratd's pipeline list and traces
`ref()` dependencies in each pipeline's SQL/Python to build a directed
graph of "what depends on what." Renders as an interactive DAG in the
portal.

## Install

```bash
docker run -d --name lineage --network infra_default --restart unless-stopped \
  -e RATD_URL=http://ratd:8080 \
  -e RATD_INTERNAL_URL=http://ratd:8090 \
  ghcr.io/squat-collective/rat-plugin-lineage:latest
```

Or uncomment the `lineage:` block in
[`infra/docker-compose.plugins.yml`](../../infra/docker-compose.plugins.yml).

Open the **Lineage** page in the portal sidebar (`/x/lineage`).

## How it works

- On startup + every 60 s, the plugin GETs `/api/v1/pipelines` and, for
  each, reads its `pipeline.sql` / `pipeline.py` from S3.
- A simple `ref("ns.layer.table")` parser identifies dependencies. The
  result is a DAG `{nodes: pipelines, edges: ref-dependencies}`.
- The portal UI is built with React-flow (the only plugin currently
  using a bundled JS toolchain — see `ui/build.mjs`).

## Environment

| Var | Default | Purpose |
|---|---|---|
| `RATD_URL` | `http://ratd:8080` | ratd base URL |
| `RATD_INTERNAL_URL` | `http://ratd:8090` | ratd internal listener |
| `GRPC_PORT` | `50106` | Port to serve on |
| `LINEAGE_REFRESH_SECS` | `60` | DAG re-scan cadence |

## Build from source

```bash
git clone https://github.com/squat-collective/rat
cd rat/plugins/rat-plugin-lineage
make build && make run
```

Note: this plugin's `bundle.js` is generated by `ui/build.mjs` (esbuild)
because the React-flow library is large enough that hand-writing
`React.createElement` calls would be impractical. `make build` runs the
JS build before the Go build.
