# Environment Variables

This is the complete reference for every environment variable across all RAT services. Variables are organized by service and category.

---

## ratd (Go Platform)

The central API server, scheduler, and plugin host.

### Server

| Variable | Default | Description |
|---|---|---|
| `RAT_LISTEN_ADDR` | `127.0.0.1:8080` | Full address the public HTTP API binds to (host:port) |
| `PORT` | тАФ | Alternative port setting. When set, binds to `0.0.0.0:<PORT>`. `RAT_LISTEN_ADDR` takes precedence |
| `INTERNAL_LISTEN_ADDR` | `127.0.0.1:8090` | Address for the **internal** HTTP listener used for service-to-service callbacks (run status from the runner) and plugin phone-home. This listener is **unauthenticated** тАФ its security model is "the network is the perimeter". Must differ from `RAT_LISTEN_ADDR` and must never be published to the host (see ADR-019) |
| `CORS_ORIGINS` | _(none)_ | Comma-separated list of allowed CORS origins. When unset, no CORS origins are configured тАФ set this to your portal URL when the portal is served from a different origin |
| `RATE_LIMIT` | `50` req/s | On/off switch, not a tunable number. Any non-zero value enables a fixed 50 req/s per-IP limiter; set to `"0"` to disable rate limiting |
| `RAT_CONFIG` | `rat.yaml` | Path to the `rat.yaml` configuration file (optional) |
| `RAT_PPROF_ADDR` | тАФ | When set (e.g. `127.0.0.1:6060`), serves Go `net/http/pprof` on a **separate** listener for profiling. Leave unset in production |

### Authentication

| Variable | Default | Description |
|---|---|---|
| `RAT_API_KEY` | тАФ | Static API key. When set, all requests require `Authorization: Bearer <key>` |

<Callout type="info">
When `RAT_API_KEY` is not set, the API is unauthenticated тАФ the default single-user, localhost-only mode. For production deployments, always set an API key or install an optional auth plugin.
</Callout>

### Plugins & Database Pooling

| Variable | Default | Description |
|---|---|---|
| `PLUGIN_ALLOW_LOOPBACK` | `false` | When `true`, lets plugins register with loopback (`127.0.0.1`) addresses. For local development only тАФ production plugins must use routable container addresses |
| `RAT_HEARTBEAT_POOL_ENABLED` | `true` | When `true`, ratd uses a dedicated small Postgres pool for the leader-election heartbeat so a saturated main pool can't starve it. Set to `false` to fall back to the shared pool |

### Database

| Variable | Default | Description |
|---|---|---|
| `DATABASE_URL` | тАФ | PostgreSQL connection string. Required. Format: `postgres://user:pass@host:port/dbname?sslmode=disable` |

### S3 Storage

| Variable | Default | Description |
|---|---|---|
| `S3_ENDPOINT` | тАФ | MinIO or S3-compatible endpoint (e.g., `minio:9000`). Required |
| `S3_ACCESS_KEY` | тАФ | S3 access key ID. Required |
| `S3_SECRET_KEY` | тАФ | S3 secret access key. Required |
| `S3_BUCKET` | `rat` | S3 bucket name where all data is stored |
| `S3_USE_SSL` | `false` | Enable TLS for S3 connections. Set to `"true"` for AWS S3 or TLS-enabled MinIO |
| `S3_METADATA_TIMEOUT` | `10s` | Timeout for S3 metadata operations (list, head, stat). Go duration format |
| `S3_DATA_TIMEOUT` | `60s` | Timeout for S3 data operations (get, put, upload). Go duration format |

### Service Discovery

| Variable | Default | Description |
|---|---|---|
| `RUNNER_ADDR` | тАФ | Runner service gRPC address. Required. Format: `http://runner:50052` |
| `RATQ_ADDR` | тАФ | Query service gRPC address. Required. Format: `http://ratq:50051` |
| `NESSIE_URL` | тАФ | Nessie REST catalog URL. Required. Format: `http://nessie:19120/api/v1` |

### Scheduler

| Variable | Default | Description |
|---|---|---|
| `SCHEDULER_ENABLED` | `true` | Enable the built-in cron scheduler. Set to `"false"` to disable |

### TLS

| Variable | Default | Description |
|---|---|---|
| `TLS_CERT_FILE` | тАФ | Path to TLS certificate for HTTPS. When set with `TLS_KEY_FILE`, ratd serves HTTPS |
| `TLS_KEY_FILE` | тАФ | Path to TLS private key for HTTPS |
| `GRPC_TLS_CA` | тАФ | Path to CA certificate for verifying gRPC connections to runner and ratq |

## runner (Python Pipeline Executor)

Executes pipelines, writes to Iceberg, and runs quality tests.

### S3 Storage

| Variable | Default | Description |
|---|---|---|
| `S3_ENDPOINT` | `minio:9000` | MinIO or S3-compatible endpoint |
| `S3_ACCESS_KEY` | тАФ | S3 access key ID. Required |
| `S3_SECRET_KEY` | тАФ | S3 secret access key. Required |
| `S3_BUCKET` | `rat` | S3 bucket name |
| `S3_USE_SSL` | `false` | Enable TLS for S3 connections |
| `S3_SESSION_TOKEN` | тАФ | Session token for temporary AWS credentials (STS) |
| `S3_REGION` | `us-east-1` | S3 region. Required by some S3-compatible stores |

### DuckDB

| Variable | Default | Description |
|---|---|---|
| `DUCKDB_MEMORY_LIMIT` | `2GB` | Maximum memory per DuckDB connection. Supports units: `MB`, `GB` |
| `DUCKDB_THREADS` | `4` | Number of CPU threads for DuckDB query execution |

<Callout type="warning">
`DUCKDB_MEMORY_LIMIT` should be set lower than the container's memory limit. If DuckDB tries to use more memory than the container allows, the OOM killer will terminate the process. A good rule of thumb: set DuckDB to 75% of the container memory.
</Callout>

### Catalog

| Variable | Default | Description |
|---|---|---|
| `NESSIE_URL` | `http://nessie:19120/api/v1` | Nessie REST catalog endpoint |

### gRPC Server

| Variable | Default | Description |
|---|---|---|
| `GRPC_PORT` | `50052` | Port for the gRPC server |
| `GRPC_TLS_CERT` | тАФ | Path to TLS certificate for the gRPC server |
| `GRPC_TLS_KEY` | тАФ | Path to TLS private key for the gRPC server |

### Execution

| Variable | Default | Description |
|---|---|---|
| `RUN_TTL_SECONDS` | `3600` | How long completed run state is kept in memory (seconds). After expiry, the run status becomes unavailable via gRPC |
| `RUNNER_MAX_CONCURRENT` | `10` | Maximum number of pipelines executing simultaneously |
| `RUNNER_MAX_WORKERS` | `10` | Maximum worker threads in the execution thread pool |
| `RUNNER_STATE_DIR` | `/tmp/rat-runner-state` | Directory for temporary run state and intermediate files |
| `RATD_CALLBACK_URL` | тАФ | Base URL for push-based status callbacks to ratd's **internal** listener. Format: `http://ratd:8090` (the internal port, NOT the public `8080`). When set, the runner POSTs terminal run status instead of waiting for ratd to poll |
| `QUALITY_TEST_TIMEOUT_SECS` | `60` | Per-quality-test timeout (seconds). A watchdog calls `conn.interrupt()` once the deadline passes; the test is recorded as failed so the rest of the suite continues |
| `QUERY_TIMEOUT_SECS` | `60` | Per-query DuckDB timeout (seconds) for the runner's internal queries |

## portal (Next.js Web IDE)

The web-based user interface.

| Variable | Default | Description |
|---|---|---|
| `NEXT_PUBLIC_API_URL` | `http://localhost:8080` | Public API URL for browser-side requests. Embedded at **build time** |
| `API_URL` | `http://ratd:8080` | Internal API URL for server-side rendering. Read at **runtime** |

<Callout type="warning">
**Build-time vs runtime**: `NEXT_PUBLIC_API_URL` is baked into the JavaScript bundle during `next build`. Changing it requires rebuilding the portal image. `API_URL` is a runtime variable that can be changed via environment without rebuilding.
</Callout>

## Production Checklist

When deploying RAT in production, ensure these variables are set to secure values:

<Steps>

### Change all default credentials

```env filename="infra/.env"
POSTGRES_USER=ratprod
POSTGRES_PASSWORD=<strong-random-password>
S3_ACCESS_KEY=<strong-random-key>
S3_SECRET_KEY=<strong-random-secret>
```

### Set an API key

```env filename="infra/.env"
RAT_API_KEY=<your-api-key>
```

### Configure CORS for your domain

```env filename="infra/.env"
CORS_ORIGINS=https://rat.yourdomain.com
```

### Set the public API URL

```env filename="infra/.env"
NEXT_PUBLIC_API_URL=https://api.rat.yourdomain.com
API_URL=http://ratd:8080
```

### Enable TLS if exposing to the internet

See the [Security Hardening](/self-hosting/security-hardening) page for TLS configuration details.

</Steps>

<Callout type="error">
Never use the default credentials (`rat`/`rat`, `minioadmin`/`minioadmin`) in production. These are well-known development defaults and will be the first thing an attacker tries.
</Callout>
