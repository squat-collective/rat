# Database Schema

ratd uses PostgreSQL for all platform state. This page documents every table, its columns, relationships, and the migration system.

## Migration System

Migrations live in `platform/internal/postgres/migrations/` and are embedded into the ratd binary via Go's `embed` package. On startup, ratd runs all pending migrations in order.

### Migration Files

| Migration | Purpose |
|-----------|---------|
| `001_initial.sql` | Core tables: pipelines, runs, schedules, namespaces |
| `002_pipeline_unique_partial.sql` | Partial unique index for soft-deleted pipelines |
| `002_landing_zones.sql` | Landing zones and landing files |
| `003_audit_log.sql` | Audit log table |
| `004_pipeline_landing_zone.sql` | Link pipelines to landing zones |
| `005_drop_pipeline_landing_zone.sql` | Remove the link (replaced by triggers) |
| `006_pipeline_triggers.sql` | Pipeline trigger definitions |
| `007_run_logs.sql` | Structured run logs |
| `008_phase_profiles.sql` | Per-phase timing profiles |
| `009_pipeline_versioning.sql` | Pipeline version pinning |
| `010_pipeline_versions.sql` | Pipeline version history |
| `011_namespace_description.sql` | Namespace descriptions |
| `012_table_metadata.sql` | Table descriptions and column metadata |
| `013_landing_zone_schema.sql` | Landing zone schema definitions |
| `014_retention_settings.sql` | Configurable retention policies |
| `015_feature_flags.sql` | Feature flag storage |
| `016_plugin_catalog.sql` | Installed plugin catalog (`plugin_catalog`, `plugin_sources`, `plugin_policies`) |
| `017_plugin_config_version.sql` | Plugin config versioning |
| `018_failed_merges.sql` | Records Phase 5 merge failures (`failed_merges`) |

<Callout type="info">
Note the two `002_` migrations. This is intentional --- they were developed in parallel branches and both apply cleanly. The migration runner tracks applied migrations by filename, so both run exactly once.
</Callout>

## Core Tables

### pipelines

The central table. Every pipeline is a row here.

```sql
CREATE TABLE pipelines (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    namespace       VARCHAR(255) NOT NULL,
    layer           VARCHAR(50) NOT NULL,
    name            VARCHAR(255) NOT NULL,
    type            VARCHAR(50) NOT NULL DEFAULT 'sql',
    description     TEXT,
    owner           VARCHAR(255),
    config          JSONB,
    soft_deleted    BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `namespace` | VARCHAR | Namespace name |
| `layer` | VARCHAR | `bronze`, `silver`, or `gold` |
| `name` | VARCHAR | Pipeline name |
| `type` | VARCHAR | `sql` or `python` |
| `description` | TEXT | Human-readable description |
| `owner` | VARCHAR | Owner user ID (set when an auth/ownership plugin is active) or null |
| `config` | JSONB | Pipeline configuration |
| `soft_deleted` | BOOLEAN | Soft-delete flag |
| `created_at` | TIMESTAMPTZ | Creation timestamp |
| `updated_at` | TIMESTAMPTZ | Last update timestamp |

### runs

Every pipeline execution creates a run.

```sql
CREATE TABLE runs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    pipeline_id     UUID REFERENCES pipelines(id),
    namespace       VARCHAR(255) NOT NULL,
    layer           VARCHAR(50) NOT NULL,
    name            VARCHAR(255) NOT NULL,
    status          VARCHAR(50) NOT NULL,
    trigger         VARCHAR(50),
    started_at      TIMESTAMPTZ,
    finished_at     TIMESTAMPTZ,
    duration_ms     BIGINT,
    rows_written    BIGINT,
    error           TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key (run ID) |
| `pipeline_id` | UUID | Foreign key to pipelines |
| `status` | VARCHAR | `pending`, `running`, `success`, `failed`, `cancelled` |
| `trigger` | VARCHAR | `manual`, `schedule`, `trigger`, `webhook` |
| `started_at` | TIMESTAMPTZ | Execution start time |
| `finished_at` | TIMESTAMPTZ | Execution end time |
| `duration_ms` | BIGINT | Total duration in milliseconds |
| `rows_written` | BIGINT | Number of rows written to the target table |
| `error` | TEXT | Error message (if failed) |

### schedules

Cron-based pipeline schedules.

```sql
CREATE TABLE schedules (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    pipeline_id     UUID REFERENCES pipelines(id),
    cron            VARCHAR(100) NOT NULL,
    enabled         BOOLEAN DEFAULT TRUE,
    last_run_at     TIMESTAMPTZ,
    next_run_at     TIMESTAMPTZ,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

| Column | Type | Description |
|--------|------|-------------|
| `cron` | VARCHAR | Cron expression (e.g., `0 9 * * *`) |
| `enabled` | BOOLEAN | Whether the schedule is active |
| `last_run_at` | TIMESTAMPTZ | Last execution time |
| `next_run_at` | TIMESTAMPTZ | Next scheduled execution |

## Supporting Tables

### namespaces

```sql
CREATE TABLE namespaces (
    name            VARCHAR(255) PRIMARY KEY,
    description     TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### pipeline_triggers

Event-driven trigger definitions (cron, pipeline success, webhook, file pattern).

```sql
CREATE TABLE pipeline_triggers (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    pipeline_id     UUID REFERENCES pipelines(id),
    trigger_type    VARCHAR(50) NOT NULL,
    config          JSONB NOT NULL,
    enabled         BOOLEAN DEFAULT TRUE,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

| Trigger Type | Description |
|--------------|-------------|
| `cron` | Time-based (cron expression) |
| `pipeline_success` | Run when another pipeline succeeds |
| `webhook` | Run via external HTTP call |
| `file_pattern` | Run when matching files land |
| `cron_dependency` | Cron + upstream dependency check |

### quality_tests

SQL-based data quality tests.

```sql
CREATE TABLE quality_tests (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    pipeline_id     UUID REFERENCES pipelines(id),
    name            VARCHAR(255) NOT NULL,
    sql             TEXT NOT NULL,
    severity        VARCHAR(20) DEFAULT 'error',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

| Column | Type | Description |
|--------|------|-------------|
| `name` | VARCHAR | Test name |
| `sql` | TEXT | Test query (returns failing rows) |
| `severity` | VARCHAR | `error` (blocks merge) or `warn` (logs only) |

### quality_results

Results of quality test runs.

```sql
CREATE TABLE quality_results (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    run_id          UUID REFERENCES runs(id),
    test_name       VARCHAR(255) NOT NULL,
    status          VARCHAR(20) NOT NULL,
    failing_rows    BIGINT DEFAULT 0,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### run_logs

Structured log entries for each run.

```sql
CREATE TABLE run_logs (
    id              BIGSERIAL PRIMARY KEY,
    run_id          UUID REFERENCES runs(id),
    timestamp       TIMESTAMPTZ NOT NULL,
    level           VARCHAR(20) NOT NULL,
    message         TEXT NOT NULL,
    phase           VARCHAR(50)
);
```

### pipeline_versions

Version history for pipeline rollback.

```sql
CREATE TABLE pipeline_versions (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    pipeline_id         UUID REFERENCES pipelines(id),
    version_number      INTEGER NOT NULL,
    message             TEXT,
    published_versions  JSONB NOT NULL,
    created_at          TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### table_metadata

User-defined descriptions for tables and columns.

```sql
CREATE TABLE table_metadata (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    namespace       VARCHAR(255) NOT NULL,
    layer           VARCHAR(50) NOT NULL,
    table_name      VARCHAR(255) NOT NULL,
    description     TEXT,
    column_metadata JSONB,
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### landing_zones & landing_files

```sql
CREATE TABLE landing_zones (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    namespace       VARCHAR(255) NOT NULL,
    name            VARCHAR(255) NOT NULL,
    description     TEXT,
    schema_json     JSONB,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE landing_files (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    zone_id         UUID REFERENCES landing_zones(id),
    filename        VARCHAR(500) NOT NULL,
    size_bytes      BIGINT,
    processed       BOOLEAN DEFAULT FALSE,
    uploaded_at     TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### audit_log

Records every mutating HTTP request (`POST`, `PUT`, `DELETE`).

```sql
CREATE TABLE audit_log (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id     VARCHAR(255) NOT NULL DEFAULT 'anonymous',
    action      VARCHAR(50)  NOT NULL,
    resource    VARCHAR(100) NOT NULL,
    detail      TEXT DEFAULT '',
    ip          VARCHAR(45),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

| Column | Type | Description |
|--------|------|-------------|
| `user_id` | VARCHAR | Acting user; `'anonymous'` when no auth plugin is active |
| `action` | VARCHAR | Lowercase HTTP method (`post`, `put`, `delete`) |
| `resource` | VARCHAR | Request path |
| `detail` | TEXT | Reserved for future use |
| `ip` | VARCHAR | Client IP |
| `created_at` | TIMESTAMPTZ | When the action occurred |

### failed_merges

Records pipeline runs whose Phase 5 merge to `main` failed (data was written and quality-tested on a branch but couldn't merge). The reaper protects these branches from the orphan-branch sweeper for a retention window so an operator can recover.

```sql
CREATE TABLE failed_merges (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    run_id          UUID NOT NULL,
    namespace       VARCHAR(255) NOT NULL,
    branch          VARCHAR(255) NOT NULL,
    reason          TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

## Plugin Tables

These tables back the plugin system (catalog, install sources, and governance policies).

### plugin_catalog

The catalog of installed plugins and their current configuration/status. The `config` column is the source of truth for plugins that use the plugin-config mechanism.

```sql
CREATE TABLE plugin_catalog (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name            VARCHAR(255) NOT NULL,
    slot            VARCHAR(100),
    status          VARCHAR(50) NOT NULL DEFAULT 'unknown',
    config          JSONB,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### plugin_sources

Configured sources (registries) from which plugins can be installed.

```sql
CREATE TABLE plugin_sources (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name            VARCHAR(255) NOT NULL,
    url             TEXT NOT NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### plugin_policies

Governance policies controlling which plugins are allowed.

```sql
CREATE TABLE plugin_policies (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name            VARCHAR(255) NOT NULL,
    rule            JSONB NOT NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

<Callout type="info">
RAT is fully free and open-source — there are no edition-gated tables. Multi-user, sharing, and ownership are provided by optional plugins; when active they manage their own resource grants rather than dedicated core tables.
</Callout>

## Feature Flags

`015_feature_flags.sql` seeds a `feature_flags` JSONB row in `platform_settings`. The defaults are:

```json
{
  "pipeline_preview": true,
  "quality_tests": true,
  "landing_zones": true,
  "pipeline_triggers": true,
  "pipeline_versions": true,
  "query_engine": true,
  "audit_log": true,
  "namespace_quotas": false,
  "distributed_rate_limiting": false
}
```

`namespace_quotas` and `distributed_rate_limiting` default to `false` because they depend on optional plugins.

## Leader Election

ratd uses PostgreSQL advisory locks for leader election. When multiple ratd instances run, only one becomes the leader and runs the scheduler and reaper.

```sql filename="Advisory lock acquisition"
SELECT pg_try_advisory_lock($1);
```

| Lock ID | Purpose |
|---------|---------|
| `1` | Scheduler leader |
| `2` | Reaper leader |

The leader holds the lock for the duration of its session. If the leader crashes, the lock is automatically released and another instance acquires it.

## Connection Pooling

ratd uses `pgxpool` for connection pooling:

| Setting | Default | Description |
|---------|---------|-------------|
| Max connections | 25 | Maximum pool size |
| Min connections | 2 | Minimum idle connections |
| Max connection lifetime | 1 hour | Recycle connections after this |
| Health check period | 1 minute | Interval for connection health checks |

<Callout type="info">
A dedicated small heartbeat pool is used for leader-election pings (controlled by `RAT_HEARTBEAT_POOL_ENABLED`) so a saturated main pool can't starve the heartbeat. Pool saturation is exposed via the `ratd_postgres_pool_*` and `ratd_postgres_heartbeat_pool_*` metrics.
</Callout>

## Backup Considerations

<Callout type="warning">
The Postgres database contains all platform state but **no actual data**. Your data lives in MinIO (Iceberg tables). A complete backup requires both Postgres (this schema) and MinIO (the data files).
</Callout>

See the [Backup & Restore](/self-hosting/backup-restore) guide for backup procedures.

---

## Reaper Defaults

The reaper enforces retention. Two of its timing defaults:

| Setting | Default | Description |
|---------|---------|-------------|
| `StuckRunTimeoutMinutes` | `30` | A `RUNNING` run older than this is force-failed |
| `ReaperIntervalMinutes` | `15` | How often the reaper tick runs |

See [Retention](/reference/api/retention) for the full set of configurable retention settings.
