# TypeScript SDK

The `@squat-collective/rat-client` SDK provides a type-safe TypeScript client for interacting with the RAT API. It is used internally by the portal and can be used in custom integrations, scripts, and applications.

---

## Installation

```bash filename="Terminal"
npm install @squat-collective/rat-client
```

<Callout type="info">
The SDK is published as an ESM + CJS dual-format package with full TypeScript declarations. It works with Node.js 20+, modern bundlers (webpack, vite, esbuild), and Deno.
</Callout>

---

## Creating a Client

```typescript filename="example.ts"
import { RatClient } from '@squat-collective/rat-client'

const client = new RatClient({
  apiUrl: 'http://localhost:8080',
})
```

### Client Options

`RatClient` takes a `RatClientOptions` object (all fields optional):

| Option | Type | Default | Description |
|---|---|---|---|
| `apiUrl` | `string` | `http://localhost:8080` | The RAT API base URL |
| `headers` | `Record<string, string>` | — | Extra headers sent on every request (use for `Authorization: Bearer <key>`) |
| `timeout` | `number` | — | Request timeout in milliseconds |
| `maxRetries` | `number` | — | Maximum retry attempts for idempotent requests |
| `onRequest` | `RequestInterceptor` | — | Request interceptor (modify headers, log, etc.) |
| `onResponse` | `ResponseInterceptor` | — | Response interceptor |

### With Authentication

Authentication uses a Bearer token in the `Authorization` header (only needed when an auth plugin or `RAT_API_KEY` is configured):

```typescript filename="example.ts"
const client = new RatClient({
  apiUrl: 'http://localhost:8080',
  headers: { Authorization: 'Bearer your-api-key-here' },
})
```

### With Custom Options

```typescript filename="example.ts"
const client = new RatClient({
  apiUrl: 'http://localhost:8080',
  headers: { Authorization: `Bearer ${process.env.RAT_API_KEY}` },
  timeout: 60_000,  // 60 seconds
  maxRetries: 5,
  onRequest: (req) => {
    console.log(`→ ${req.method} ${req.url}`)
    return req
  },
  onResponse: (res) => {
    console.log(`← ${res.status}`)
    return res
  },
})
```

## Resource Namespaces

The client organizes API methods into resource namespaces (`health`, `pipelines`, `runs`, `query`, `tables`, `storage`, `namespaces`, `landing`, `triggers`, `quality`, `lineage`, `retention`, `plugins`), each mapping to a group of related endpoints.

### health

Check platform health and feature availability.

```typescript
// Health check — returns the service status
const health = await client.health.getHealth()
// { status: 'ok' }

// Active features and loaded plugins
const features = await client.health.getFeatures()
// { edition: 'community', namespaces: false, multi_user: false, landing_zones: true, plugins: {} }
```

### pipelines

Create, read, update, and delete pipelines.

```typescript
// List pipelines (optional filters)
const pipelines = await client.pipelines.list()
const filtered = await client.pipelines.list({ namespace: 'ecommerce', layer: 'silver' })

// Get a specific pipeline
const pipeline = await client.pipelines.get('ecommerce', 'silver', 'clean_orders')

// Create a pipeline
const created = await client.pipelines.create({
  namespace: 'ecommerce',
  layer: 'silver',
  name: 'clean_orders',
})

// Update a pipeline
const updated = await client.pipelines.update('ecommerce', 'silver', 'clean_orders', {
  description: 'Updated description',
})

// Delete a pipeline
await client.pipelines.delete('ecommerce', 'silver', 'clean_orders')

// Publish a draft, then roll back to an earlier version
await client.pipelines.publish('ecommerce', 'silver', 'clean_orders', 'ship v2')
await client.pipelines.rollback('ecommerce', 'silver', 'clean_orders', 1)
```

### runs

Trigger pipeline runs and monitor their status.

```typescript
// Trigger a new run — note `pipeline` (not `name`); returns { run_id, status }
const created = await client.runs.create({
  namespace: 'ecommerce',
  layer: 'silver',
  pipeline: 'clean_orders',
  trigger: 'manual',
})

// Get run status (the Run object's id is `created.run_id`)
const run = await client.runs.get(created.run_id)

// List runs (optional filters)
const { runs, total } = await client.runs.list({
  namespace: 'ecommerce',
  layer: 'silver',
  pipeline: 'clean_orders',
})

// Cancel a running pipeline → { run_id, status }
await client.runs.cancel(created.run_id)

// Fetch the run's logs (returns { logs, status }, not a stream)
const { logs } = await client.runs.logs(created.run_id)
for (const log of logs) {
  console.log(`[${log.level}] ${log.message}`)
}
```

### query

Execute interactive DuckDB queries (read-only).

```typescript
// Execute a SQL query
const result = await client.query.execute({
  sql: 'SELECT * FROM default.silver.clean_orders LIMIT 10',
  namespace: 'ecommerce',
  limit: 1000,
})
// rows are objects keyed by column name:
// { columns: [{ name: 'order_id', type: 'VARCHAR' }, ...],
//   rows: [{ order_id: 'ORD-001', amount: 129.99 }, ...],
//   total_rows: 10, duration_ms: 42 }
```

<Callout type="info">
Schema introspection and pipeline preview live on other resources, not `query`: use `client.tables.schema(...)` and `client.tables.preview(...)` (or `client.pipelines.preview(...)` for a dry-run).
</Callout>

### tables

Interact with Iceberg tables.

```typescript
// List tables (optional namespace/layer filters)
const tables = await client.tables.list({ namespace: 'ecommerce' })

// Get table detail + metadata
const detail = await client.tables.get('ecommerce', 'silver', 'clean_orders')

// Read the table schema
const schema = await client.tables.schema({ namespace: 'ecommerce', layer: 'silver' })

// Preview rows from a table (returns a QueryResult)
const preview = await client.tables.preview('ecommerce', 'silver', 'clean_orders')

// Update table documentation
await client.tables.updateMetadata('ecommerce', 'silver', 'clean_orders', {
  description: 'Cleaned orders',
  column_descriptions: { order_id: 'Unique order identifier' },
})
```

### storage

Manage files in S3 (pipeline code, etc.).

```typescript
// List files under a prefix
const files = await client.storage.list('ecommerce/pipelines/silver/clean_orders/')

// Read a file
const content = await client.storage.read('ecommerce/pipelines/silver/clean_orders/pipeline.sql')

// Write a file
await client.storage.write('ecommerce/pipelines/silver/clean_orders/pipeline.sql', sqlContent)

// Delete a file
await client.storage.delete('ecommerce/pipelines/silver/clean_orders/old_file.sql')

// Upload a file (multipart) — file is a File/Blob
await client.storage.upload('ecommerce/landing/orders/data.csv', fileBlob, 'data.csv')
```

### namespaces

Manage namespaces (organizational containers for pipelines).

```typescript
// List all namespaces
const namespaces = await client.namespaces.list()

// Create a namespace (takes a string, not an object)
const ns = await client.namespaces.create('ecommerce')

// Update / delete
await client.namespaces.update('ecommerce', { description: 'Storefront data' })
await client.namespaces.delete('ecommerce')
```

### landing

Manage landing zones (upload targets for external data).

```typescript
// List landing zones (optional namespace filter)
const zones = await client.landing.list({ namespace: 'ecommerce' })

// List files in a landing zone (namespace, zone name)
const files = await client.landing.listFiles('ecommerce', 'orders')

// Upload a file to a landing zone — uploadFile(ns, zone, file, filename?)
await client.landing.uploadFile('ecommerce', 'orders', fileBlob, 'batch_001.csv')
```

### triggers

Manage pipeline triggers (cron, event-driven, dependency-aware). Trigger methods are scoped to a pipeline by `(namespace, layer, name)`.

```typescript
// List triggers for a pipeline
const triggers = await client.triggers.list('ecommerce', 'silver', 'clean_orders')

// Create a cron trigger
const trigger = await client.triggers.create('ecommerce', 'silver', 'clean_orders', {
  type: 'cron',
  config: { cron_expr: '0 9 * * *' },
  enabled: true,
})

// Update / delete a trigger by its id
await client.triggers.update('ecommerce', 'silver', 'clean_orders', trigger.id, { enabled: false })
await client.triggers.delete('ecommerce', 'silver', 'clean_orders', trigger.id)
```

### quality

Manage quality tests for pipelines.

```typescript
// List quality tests for a pipeline
const tests = await client.quality.list('ecommerce', 'silver', 'clean_orders')

// Create a quality test
await client.quality.create('ecommerce', 'silver', 'clean_orders', {
  name: 'no_nulls',
  sql: 'SELECT * FROM {{ this }} WHERE order_id IS NULL',
  severity: 'error',
})

// Run the pipeline's quality tests now
const result = await client.quality.run('ecommerce', 'silver', 'clean_orders')

// Delete a quality test
await client.quality.delete('ecommerce', 'silver', 'clean_orders', 'no_nulls')
```

### lineage

Explore the data lineage graph.

```typescript
// Get the lineage graph (optional namespace filter) → { nodes, edges }
const graph = await client.lineage.get({ namespace: 'ecommerce' })
```

<Callout type="info">
Lineage is served by the optional `lineage` plugin. `client.lineage.get()` returns `404` when that plugin isn't installed.
</Callout>

### retention

Manage data retention policies.

```typescript
// Get / update system retention config
const config = await client.retention.getConfig()
await client.retention.updateConfig({
  stuck_run_timeout_minutes: 30,
  reaper_interval_minutes: 15,
})

// Reaper status + manual trigger
const status = await client.retention.getStatus()
await client.retention.triggerRun()
```

## Error Handling

The SDK throws typed errors that map to HTTP status codes. All errors extend the base `RatError` class.

```typescript
import {
  RatError,
  AuthenticationError,
  AuthorizationError,
  NotFoundError,
  ConflictError,
  ValidationError,
  ServerError,
  ConnectionError,
} from '@squat-collective/rat-client'
```

### Error Types

| Error Class | HTTP Status | When |
|---|---|---|
| `AuthenticationError` | 401 | Missing or invalid API key / token (only when auth is enabled) |
| `AuthorizationError` | 403 | Valid credentials but insufficient permissions (only with the enforcement plugin) |
| `NotFoundError` | 404 | Resource does not exist |
| `ConflictError` | 409 | Resource already exists or version conflict |
| `ValidationError` | 400/422 | Invalid request body (missing fields, bad types) |
| `ServerError` | 500+ | Internal server error |
| `ConnectionError` | — | Network error, timeout, DNS failure |

### Error Properties

Every `RatError` has these properties:

```typescript
interface RatError extends Error {
  message: string        // Human-readable error message
  status: number         // HTTP status code (0 for ConnectionError)
  code: string           // Machine-readable error code (e.g., 'NOT_FOUND')
  details?: unknown      // Additional error details from the API
}
```

### Example Usage

```typescript
import { RatClient, NotFoundError, ValidationError } from '@squat-collective/rat-client'

const client = new RatClient({ apiUrl: 'http://localhost:8080' })

try {
  const pipeline = await client.pipelines.get('ecommerce', 'silver', 'clean_orders')
} catch (error) {
  if (error instanceof NotFoundError) {
    console.log('Pipeline does not exist yet')
  } else if (error instanceof ValidationError) {
    console.log('Invalid request:', error.details)
  } else {
    throw error // unexpected error, rethrow
  }
}
```

## Transport Configuration

### Retries

The SDK automatically retries **idempotent** requests on transient failures. Only `GET`, `PUT`, and `DELETE` requests are retried. `POST` requests are **never** retried to avoid duplicate side effects.

| Setting | Default | Description |
|---|---|---|
| Retry count | 3 | Maximum number of retry attempts |
| Backoff strategy | Exponential | 500ms, 1s, 2s, 4s, ... up to 10s |
| Minimum delay | 500ms | Shortest wait between retries |
| Maximum delay | 10s | Longest wait between retries |
| Retried methods | `GET`, `PUT`, `DELETE` | `POST` is never retried |
| Retried status codes | 408, 429, 502, 503, 504 | Transient server errors |

### Timeout

All requests have a default timeout of **30 seconds**. Override per-client:

```typescript
const client = new RatClient({
  apiUrl: 'http://localhost:8080',
  timeout: 60_000, // 60 seconds
})
```

### Request/Response Interceptors

Interceptors let you modify requests before they are sent and responses before they are processed. Common use cases: logging, adding custom headers, metrics.

```typescript
const client = new RatClient({
  apiUrl: 'http://localhost:8080',
  onRequest: (request) => {
    request.headers.set('X-Request-ID', crypto.randomUUID())
    console.log(`→ ${request.method} ${request.url}`)
    return request
  },
  onResponse: (response) => {
    console.log(`← ${response.status} (${response.headers.get('x-response-time')})`)
    return response
  },
})
```

## Common Patterns

### Poll for Run Completion

```typescript
async function waitForRun(client: RatClient, runId: string): Promise<Run> {
  while (true) {
    const run = await client.runs.get(runId)
    if (run.status === 'success' || run.status === 'failed' || run.status === 'cancelled') {
      return run
    }
    await new Promise((resolve) => setTimeout(resolve, 2000))
  }
}

// Usage
const created = await client.runs.create({
  namespace: 'ecommerce',
  layer: 'silver',
  pipeline: 'clean_orders',
  trigger: 'manual',
})
const result = await waitForRun(client, created.run_id)
console.log(`Run ${result.status}: ${result.duration_ms}ms`)
```

### Trigger and Read Logs

```typescript
const created = await client.runs.create({
  namespace: 'ecommerce',
  layer: 'silver',
  pipeline: 'clean_orders',
  trigger: 'manual',
})

const { logs } = await client.runs.logs(created.run_id)
for (const log of logs) {
  console.log(`[${new Date(log.timestamp).toISOString()}] [${log.level}] ${log.message}`)
}
```

### Bulk Pipeline Discovery

```typescript
// List all pipelines across all namespaces
const { pipelines } = await client.pipelines.list()

// Group by layer
const byLayer = Object.groupBy(pipelines, (p) => p.layer)
console.log(`Bronze: ${byLayer.bronze?.length ?? 0}`)
console.log(`Silver: ${byLayer.silver?.length ?? 0}`)
console.log(`Gold: ${byLayer.gold?.length ?? 0}`)
```

### Upload Data to a Landing Zone

```typescript
import { readFileSync } from 'fs'

const csv = new Blob([readFileSync('./orders_2026_02.csv')])
await client.landing.uploadFile('ecommerce', 'orders', csv, 'orders_2026_02.csv')

// Trigger the Bronze ingestion pipeline
await client.runs.create({
  namespace: 'ecommerce',
  layer: 'bronze',
  pipeline: 'raw_orders',
  trigger: 'manual',
})
```

## SDK Development

The SDK source lives at `sdk-typescript/` in the monorepo. To build and test:

```bash filename="Terminal"
# Build the SDK (ESM + CJS + DTS)
make sdk-build

# Run SDK tests
make sdk-test
```

The SDK is built with `tsup` and produces three outputs:
- `dist/index.mjs` — ESM module
- `dist/index.cjs` — CommonJS module
- `dist/index.d.ts` — TypeScript declarations
