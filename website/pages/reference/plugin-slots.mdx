# Plugin Slots Reference

The portal plugin slot system allows external packages to inject React components into predefined locations in the UI.

## `PluginSlot` Component

Renders all components registered for a named slot.

### Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `name` | `string` | Yes | The slot name to render components for |
| `fallback` | `React.ReactNode` | No | Fallback content when no `PluginRegistryProvider` is present |
| `...props` | `Record<string, unknown>` | No | Extra props passed through to each slot component |

### Usage

```tsx
import { PluginSlot } from "@/components/plugins";

// Basic slot
<PluginSlot name="main-header" />

// With passthrough props
<PluginSlot name="sidebar-user" collapsed={collapsed} />

// With fallback
<PluginSlot name="sidebar-user" fallback={<DefaultUserIcon />} />
```

### Behavior

- **No provider**: Renders `fallback` if provided, otherwise renders nothing
- **No components registered**: Renders nothing
- **Components registered**: Renders all components in order, passing extra props to each

## `PluginRegistryProvider`

Provides the plugin registry to all `PluginSlot` components in the tree.

### Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `registry` | `PluginRegistry` | Yes | `{ slots, navItems, routes }` |
| `children` | `React.ReactNode` | Yes | Child components |

### Usage

```tsx
import { PluginRegistryProvider } from "@/components/plugins";

const registry = {
  slots: {
    "sidebar-user": [UserMenu],
    "main-header": [NamespacePicker],
  },
  navItems: [],
  routes: [],
};

<PluginRegistryProvider registry={registry}>
  <App />
</PluginRegistryProvider>
```

## `usePluginRegistry` Hook

Returns the current plugin registry, or `null` if no provider is present.

```tsx
import { usePluginRegistry } from "@/components/plugins";

function MyComponent() {
  const registry = usePluginRegistry();
  // registry is PluginRegistry | null
}
```

## Types

```typescript
/** A React component that can render in a plugin slot. */
type SlotComponent<P = Record<string, unknown>> = React.ComponentType<P>;

/** A navigation item injected by a plugin into the sidebar. */
type PluginNavItem = {
  label: string;
  icon: string;   // lucide icon name
  href: string;
  priority: number;
};

/** A full-page route registered by a plugin (mounted under /x/...). */
type PluginRoute = {
  path: string;   // e.g. "/x/my-plugin"
  component: React.ComponentType<{ path: string[] }>;
};

/** Merged registry from all loaded plugins. */
type PluginRegistry = {
  slots: Record<string, SlotComponent<any>[]>;
  navItems: PluginNavItem[];
  routes: PluginRoute[];
};
```

<Callout type="info">
`PluginRegistry` is **not** a flat map — it is an object with `slots`, `navItems`, and `routes`. The `PluginRegistryProvider` accepts this shape; `<PluginSlot>` reads from `registry.slots`.
</Callout>

## Default Behavior

When no plugins are installed, all slots render nothing — zero overhead in the bundle. There is no build-time plugin configuration: plugins are discovered and loaded at runtime.

When a plugin is installed and enabled, the portal fetches the enabled-plugin list at runtime and injects each plugin's JS bundle. Each bundle calls `window.__RAT_REGISTER_PLUGIN(name, registration)` to register its slot components, nav items, and routes, which the loader merges into the `PluginRegistry`. See [Plugin UI Development](/contributing/plugin-ui) for how to build plugin bundles.
